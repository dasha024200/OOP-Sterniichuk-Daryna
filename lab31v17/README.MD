Лабораторна робота №31 варіант 17.
 Реалізовано EventService із залежностями IEventRepository та ICalendarService, які передаються через конструктор. Написано 8 unit-тестів із використанням бібліотеки Moq. У тестах застосовано Setup для налаштування поведінки mock-об’єктів та Verify для перевірки викликів методів. Усі тести проходять успішно.

Moq — це бібліотека для створення mock-об’єктів під час тестування залежностей.

Setup використовується для задання очікуваної поведінки mock-об’єкта.

Verify перевіряє, чи був викликаний потрібний метод потрібну кількість разів.

Навіщо DI через конструктор?
Щоб можна було підміняти реальні залежності mock-об’єктами в тестах.
