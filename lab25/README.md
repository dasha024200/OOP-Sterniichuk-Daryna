Лабораторна робота №25
Тема: Інтеграція патернів проєктування
У цій лабораторній роботі реалізовано інтегровану систему, яка поєднує чотири патерни проєктування: Factory Method, Singleton, Strategy та Observer.
У роботі реалізовано:
Factory Method – для створення різних типів логерів (ConsoleLogger та FileLogger).
Singleton – клас LoggerManager, який гарантує єдиний екземпляр менеджера логування.
Strategy – для зміни алгоритму обробки даних (EncryptDataStrategy та CompressDataStrategy).
Observer – для сповіщення про обробку даних через подію DataProcessed.
Програма містить три сценарії:
Повна інтеграція – логування відбувається в консоль, використовуєтьсястратегія шифрування.
Динамічна зміна логера – після першої обробки логер змінюється на файловий.
Динамічна зміна стратегії – стратегія змінюється на компресію даних.
Програма компілюється та запускається без помилок.
Усі патерни працюють разом та демонструють взаємодію між компонентами системи.
Висновок:
У ході роботи було продемонстровано інтеграцію кількох патернів проєктування в одній системі, що забезпечує гнучкість, розширюваність та правильну архітектуру програмного забезпечення.
